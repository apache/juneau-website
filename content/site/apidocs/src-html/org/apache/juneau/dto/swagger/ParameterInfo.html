021/**
022 * Describes a single operation parameter.
023 * <p>
024 * A unique parameter is defined by a combination of a name and location.
025 * <p>
026 * There are five possible parameter types.
027 * <ul>
028 *    <li><js>"path"</js> - Used together with Path Templating, where the parameter value is actually part of the operation's URL.
029 *       This does not include the host or base path of the API.
030 *       For example, in <code>/items/{itemId}</code>, the path parameter is <code>itemId</code>.
031 *    <li><js>"query"</js> - Parameters that are appended to the URL.
032 *       For example, in <code>/items?id=###</code>, the query parameter is <code>id</code>.
033 *    <li><js>"header"</js> - Custom headers that are expected as part of the request.
034 *    <li><js>"body"</js> - The payload that's appended to the HTTP request.
035 *       Since there can only be one payload, there can only be one body parameter.
036 *       The name of the body parameter has no effect on the parameter itself and is used for documentation purposes only.
037 *       Since Form parameters are also in the payload, body and form parameters cannot exist together for the same operation.
038 *    <li><js>"formData"</js> - Used to describe the payload of an HTTP request when either <code>application/x-www-form-urlencoded</code>, <code>multipart/form-data</code> or both are used as the content type of the request (in Swagger's definition, the consumes property of an operation).
039 *       This is the only parameter type that can be used to send files, thus supporting the file type.
040 *       Since form parameters are sent in the payload, they cannot be declared together with a body parameter for the same operation.
041 *       Form parameters have a different format based on the content-type used (for further details, consult <code>http://www.w3.org/TR/html401/interact/forms.html#h-17.13.4</code>):
042 *       <ul>
043 *          <li><js>"application/x-www-form-urlencoded"</js> - Similar to the format of Query parameters but as a payload.
044 *             For example, <code>foo=1&bar=swagger</code> - both <code>foo</code> and <code>bar</code> are form parameters.
045 *             This is normally used for simple parameters that are being transferred.
046 *          <li><js>"multipart/form-data"</js> - each parameter takes a section in the payload with an internal header.
047 *             For example, for the header <code>Content-Disposition: form-data; name="submit-name"<code> the name of the parameter is <code>submit-name</code>.
048 *             This type of form parameters is more commonly used for file transfers.
049 *       </ul>
050 *    </li>
051 * </ul>
052 */
