069/**
070 * Utility class for interfacing with remote REST interfaces.
071 *
072 * <p class='w900'>
073 * Built upon the feature-rich Apache HttpClient library, the Juneau RestClient API adds support for fluent-style
074 * REST calls and the ability to perform marshalling of POJOs to and from HTTP parts.
075 *
076 * <h5 class='figure'>Example:</h5>
077 * <p class='bcode w800'>
078 *    <jc>// Create a basic REST client with JSON support and download a bean.</jc>
079 *    <jk>try</jk> (RestClient c = RestClient.<jsm>create</jsm>().build()) {
080 *       MyBean b = c.get(<jsf>URL</jsf>).run().assertStatusCode(200).getBody().as(MyBean.<jk>class</jk>);
081 *    }
082 * </p>
083 *
084 * <p class='w900'>
085 * Breaking apart the fluent call, we can see the classes being used:
086 * <p class='bcode w800'>
087 *    <jc>// Create a basic REST client with JSON support and download a bean.</jc>
088 *    <jk>try</jk> (RestClient c = RestClient.<jsm>create</jsm>().build()) {
089 *       RestRequest req = c.get(<jsf>URL</jsf>);
090 *       RestResponse res = req.run();
091 *       res.assertStatusCode(200);
092 *       RestResponseBody body = res.getBody();
093 *       MyBean b = body.as(MyBean.<jk>class</jk>);
094 *    }
095 * </p>
096 *
097 * <p class='w900'>
098 * It additionally provides support for creating remote proxy interfaces using REST as the transport medium.
099 *
100 * <h5 class='figure'>Example:</h5>
101 * <p class='bcode w800'>
102 *    <jc>// Define a Remote proxy for interacting with a REST interface.</jc>
103 *    <ja>@Remote</ja>(path=<js>"/petstore"</js>)
104 *    <jk>public interface</jk> PetStore {
105 *       <ja>@RemoteMethod</ja>(httpMethod=<jsf>POST</jsf>, path=<js>"/pets"</js>)
106 *       Pet addPet(
107 *          <ja>@Body</ja> CreatePet pet,
108 *          <ja>@Header</ja>(<js>"E-Tag"</js>) UUID etag,
109 *          <ja>@Query</ja>(<js>"debug"</js>) <jk>boolean</jk> debug
110 *       );
111 *    }
112 *
113 *    <jc>// Use a RestClient with default Simple JSON support.</jc>
114 *    <jk>try</jk> (RestClient c = RestClient.<jsm>create</jsm>().simpleJson().build()) {
115 *       PetStore store = c.getRemote(PetStore.<jk>class</jk>, <js>"http://localhost:10000"</js>);
116 *       CreatePet cp = <jk>new</jk> CreatePet(<js>"Fluffy"</js>, 9.99);
117 *       Pet p = store.addPet(cp, UUID.<jsm>randomUUID</jsm>(), <jk>true</jk>);
118 *    }
119 * </p>
120 *
121 * <p class='w900'>
122 * The classes are closely tied to Apache HttpClient, yet provide lots of additional functionality:
123 * <ul class='javatree'>
124 *    <li class='jc'>{@link RestClient} <jk>extends</jk> {@link HttpClient}, creates {@link RestRequest} objects.
125 *    <li class='jc'>{@link RestRequest} <jk>extends</jk> {@link HttpUriRequest}, creates {@link RestResponse} objects.
126 *    <li class='jc'>{@link RestResponse} <jk>extends</jk> {@link HttpResponse}, creates {@link RestResponseBody} and {@link RestResponseHeader} objects.
127 *    <li class='jc'>{@link RestResponseBody} <jk>extends</jk> {@link HttpEntity}
128 *    <li class='jc'>{@link RestResponseHeader} <jk>extends</jk> {@link Header}
129 * </ul>
130 *
131 *
132 * <h4 class='topic'>Instantiation and Lifecycle</h4>
133 *
134 * <p class='w900'>
135 * Instances of this class are built using the {@link RestClientBuilder} class which can be constructed using
136 * the {@link #create() RestClient.create()} method as shown above.
137 *
138 * <p class='w900'>
139 * Clients are typically created with a root URL so that relative URLs can be used when making requests.
140 * This is done using the {@link RestClientBuilder#rootUrl(Object)} method.
141 *
142 * <h5 class='figure'>Example:</h5>
143 * <p class='bcode w800'>
144 *    <jc>// Create a client where all URLs are relative to localhost.</jc>
145 *    RestClient c = RestClient.<jsm>create</jsm>().rootUrl(<js>"http://localhost:5000"</js>).build();
146 *
147 *    <jc>// Use relative paths.</jc>
148 *    String body = c.get(<js>"/subpath"</js>).run().getBody().asString();
149 * </p>
150 *
151 * <p class='w900'>
152 * The {@link RestClient} class creates {@link RestRequest} objects using the following methods:
153 *
154 * <ul class='javatree'>
155 *    <li class='jc'>{@link RestClient}
156 *    <ul>
157 *       <li class='jm'>{@link RestClient#get(Object) get(Object url)}
158 *       <li class='jm'>{@link RestClient#put(Object,Object) put(Object url, Object body)}
159 *       <li class='jm'>{@link RestClient#put(Object) put(Object url)}
160 *       <li class='jm'>{@link RestClient#post(Object) post(Object url, Object body)}
161 *       <li class='jm'>{@link RestClient#post(Object) post(Object url)}
162 *       <li class='jm'>{@link RestClient#patch(Object,Object) patch(Object url, Object body)}
163 *       <li class='jm'>{@link RestClient#patch(Object) patch(Object url)}
164 *       <li class='jm'>{@link RestClient#delete(Object) delete(Object url)}
165 *       <li class='jm'>{@link RestClient#options(Object) options(Object url)}
166 *       <li class='jm'>{@link RestClient#formPost(Object,Object) formPost(Object url, Object body)}
167 *       <li class='jm'>{@link RestClient#formPost(Object) formPost(Object url)}
168 *       <li class='jm'>{@link RestClient#formPost(Object,NameValuePairs) formPost(Object url, NameValuePairs parameters)}
169 *       <li class='jm'>{@link RestClient#formPost(Object,NameValuePair...) formPost(Object url, NameValuePair...parameters)}
170 *       <li class='jm'>{@link RestClient#formPost(Object,Object...) formPost(Object url, Object...parameters)}
171 *       <li class='jm'>{@link RestClient#request(HttpMethod,Object,Object) request(HttpMethod method, Object url, Object body)}
172 *    </ul>
173 * </ul>
174 *
175 * <p class='w900'>
176 * The {@link RestRequest} class creates {@link RestResponse} objects using the following methods:
177 *
178 * <ul class='javatree'>
179 *    <li class='jc'>{@link RestRequest}
180 *    <ul>
181 *       <li class='jm'>{@link RestRequest#run() run()}
182 *       <li class='jm'>{@link RestRequest#execute() execute()}
183 *    </ul>
184 * </ul>
185 *
186 * <p class='w900'>
187 * The distinction between the two methods is that {@link RestRequest#execute() execute()} automatically consumes the response body and
188 * {@link RestRequest#run() run()} does not.  Note that you must consume response bodies in order for HTTP connections to be freed up
189 * for reuse!  The {@link InputStream InputStreams} returned by the {@link RestResponseBody} object are auto-closing once
190 * they are exhausted, so it is often not necessary to explicitly close them.
191 *
192 * <p class='w900'>
193 * The following examples show the distinction between the two calls:
194 *
195 * <p class='bcode w800'>
196 *    <jc>// Consuming the response, so use run().</jc>
197 *    String body = client.get(<jsf>URL</jsf>).run().getBody().asString();
198 *
199 *    <jc>// Only interested in response status code, so use execute().</jc>
200 *   <jk>int</jk> status = client.get(<jsf>URL</jsf>).execute().getStatusCode();
201 * </p>
202 *
203 * <ul class='notes'>
204 *    <li>Don't confuse the behavior on the {@link RestRequest#execute()} method with the various execute methods defined on
205 *       the {@link RestClient} class.
206 * </ul>
207 *
208 *
209 * <h4 class='topic'>POJO Marshalling</h4>
210 *
211 * <p class='w900'>
212 * By default, JSON support is provided for HTTP request and response bodies.
213 * Other languages can be specified using any of the following builder methods:
214 * <ul class='javatree'>
215 *    <li class='jc'>{@link RestClientBuilder}
216 *    <ul>
217 *       <li class='jm'>{@link RestClientBuilder#json() json()}
218 *       <li class='jm'>{@link RestClientBuilder#simpleJson() simpleJson()}
219 *       <li class='jm'>{@link RestClientBuilder#xml() xml()}
220 *       <li class='jm'>{@link RestClientBuilder#html() html()}
221 *       <li class='jm'>{@link RestClientBuilder#plainText() plainText()}
222 *       <li class='jm'>{@link RestClientBuilder#msgpack() msgpack()}
223 *       <li class='jm'>{@link RestClientBuilder#uon() uon()}
224 *       <li class='jm'>{@link RestClientBuilder#urlEnc() urlEnc()}
225 *       <li class='jm'>{@link RestClientBuilder#openapi() openapi()}
226 *    </ul>
227 * </ul>
228 *
229 * <h5 class='figure'>Example:</h5>
230 * <p class='bcode w800'>
231 *    <jc>// Create a basic REST client with Simplified-JSON support.</jc>
232 *    <jc>// Typically easier to use when performing unit tests.</jc>
233 *    RestClient c = RestClient.<jsm>create</jsm>().simpleJson().build();
234 * </p>
235 *
236 * <p class='w900'>
237 *    Other methods are also provided for specifying the serializers and parsers used for lower-level marshalling support:
238 * <ul class='javatree'>
239 *    <li class='jc'>{@link RestClientBuilder}
240 *    <ul>
241 *       <li class='jm'>{@link RestClientBuilder#serializer(Serializer) serializer(Serializer)}
242 *       <li class='jm'>{@link RestClientBuilder#serializer(Class) serializer(Class&lt;? extends Serializer>)}
243 *       <li class='jm'>{@link RestClientBuilder#parser(Parser) parser(Parser)}
244 *       <li class='jm'>{@link RestClientBuilder#parser(Class) parser(Class&lt;? extends Parser>)}
245 *       <li class='jm'>{@link RestClientBuilder#marshall(Marshall) marshall(Marshall)}
246 *    </ul>
247 * </ul>
248 *
249 * <p class='w900'>
250 * HTTP parts (headers, query parameters, form data...) are serialized and parsed using the {@link HttpPartSerializer}
251 * and {@link HttpPartParser} APIs.  By default, clients are configured to use {@link OpenApiSerializer} and
252 * {@link OpenApiParser}.  These can be overridden using the following methods:
253 * <ul class='javatree'>
254 *    <li class='jc'>{@link RestClientBuilder}
255 *    <ul>
256 *       <li class='jm'>{@link RestClientBuilder#partSerializer(HttpPartSerializer) partSerializer(HttpPartSerializer)}
257 *       <li class='jm'>{@link RestClientBuilder#partSerializer(Class) partSerializer(Class&lt;? extends HttpPartSerializer>)}
258 *       <li class='jm'>{@link RestClientBuilder#partParser(HttpPartParser) partParser(HttpPartParser)}
259 *       <li class='jm'>{@link RestClientBuilder#partParser(Class) partParser(Class&lt;? extends HttpPartParser>)}
260 *    </ul>
261 * </ul>
262 *
263 * <p class='w900'>
264 * The {@link RestClientBuilder} class also provides convenience methods for setting common serializer and parser
265 * settings.
266 *
267 * <h5 class='figure'>Example:</h5>
268 * <p class='bcode w800'>
269 *    <jc>// Create a basic REST client with JSON support.</jc>
270 *    <jc>// Use single-quotes and whitespace.</jc>
271 *    RestClient c = RestClient.<jsm>create</jsm>().json().sq().ws().build();
272 * </p>

520 *          {@link Reader} - Raw contents of {@code Reader} will be serialized to remote resource.
521 *       <li class='jc'>

531 *    </ul>

534 *    <li>If the serializer on the client or request is explicitly set to <jk>null</jk>, POJOs will be converted to strings
535 *       using the registered part serializer as content type <js>"text/plain</js>.  If the part serializer is also <jk>null</jk>,
536 *       POJOs will be converted to strings using {@link ClassMeta#toString(Object)} which typically just calls {@link Object#toString()}.

930 * <h4 class='topic'>REST Proxies</h4>
931 *
932 * <p class='w900'>
933 * One of the more powerful features of the REST client class is the ability to produce Java interface proxies against
934 * arbitrary remote REST resources.
935 *
936 * <h5 class='figure'>Example:</h5>
937 * <p class='bcode w800'>
938 *    <jc>// Define a Remote proxy for interacting with a REST interface.</jc>
939 *    <ja>@Remote</ja>(path=<js>"/petstore"</js>)
940 *    <jk>public interface</jk> PetStore {
941 *       <ja>@RemoteMethod</ja>(httpMethod=<jsf>POST</jsf>, path=<js>"/pets"</js>)
942 *       Pet addPet(
943 *          <ja>@Body</ja> CreatePet pet,
944 *          <ja>@Header</ja>(<js>"E-Tag"</js>) UUID etag,
945 *          <ja>@Query</ja>(<js>"debug"</js>) <jk>boolean</jk> debug
946 *       );
947 *    }
948 *
949 *    <jc>// Use a RestClient with default Simple JSON support.</jc>
950 *    <jk>try</jk> (RestClient c = RestClient.<jsm>create</jsm>().simpleJson().build()) {
951 *       PetStore store = c.getRemote(PetStore.<jk>class</jk>, <js>"http://localhost:10000"</js>);
952 *       CreatePet cp = <jk>new</jk> CreatePet(<js>"Fluffy"</js>, 9.99);
953 *       Pet p = store.addPet(cp, UUID.<jsm>randomUUID</jsm>(), <jk>true</jk>);
954 *    }
955 * </p>
956 *
957 * <p class='w900'>
958 * The methods to retrieve remote interfaces are:
959 *
960 * <ul class='javatree'>
961 *    <li class='jc'>{@link RestClient}
962 *    <ul>
963 *       <li class='jm'><c>{@link RestClient#getRemote(Class) getRemote(Class&lt;T&gt;)} <jk>returns</jk> T</c>
964 *       <li class='jm'><c>{@link RestClient#getRemote(Class,Object) getRemote(Class&lt;T&gt;,Object)} <jk>returns</jk> T</c>
965 *       <li class='jm'><c>{@link RestClient#getRemote(Class,Object,Serializer,Parser) getRemote(Class&lt;T&gt;,Object,Serializer,Parser)} <jk>returns</jk> T</c>
966 *       <li class='jm'><c>{@link RestClient#getRrpcInterface(Class) getRrpcInterface(Class&lt;T&gt;)} <jk>returns</jk> T</c>
967 *       <li class='jm'><c>{@link RestClient#getRrpcInterface(Class,Object) getRrpcInterface(Class&lt;T&gt;,Object)} <jk>returns</jk> T</c>
968 *       <li class='jm'><c>{@link RestClient#getRrpcInterface(Class,Object,Serializer,Parser) getRrpcInterface(Class&lt;T&gt;,Object,Serializer,Parser)} <jk>returns</jk> T</c>
969 *    </ul>
970 * </ul>
971 *
972 * <p class='w900'>
973 * Two basic types of remote interfaces are provided:
974 *
975 * <ul class='spaced-list'>
976 *    <li>{@link Remote @Remote}-annotated interfaces.  These can be defined against arbitrary external REST resources.
977 *    <li>RPC-over-REST interfaces.  These are Java interfaces that allow you to make method calls on server-side POJOs.
978 * </ul>
979 *
980 * <p class='w900'>
981 * Refer to the following documentation on both flavors:
982 *
983 * <ul class='doctree'>
984 *    <li class='link'>{@doc juneau-rest-client.RestProxies}
985 *    <li class='link'>{@doc juneau-rest-server.restRPC}
986 * </ul>
987 *
988 * <br>
989 * <hr class='w900'>
